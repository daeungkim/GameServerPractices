# GameServerPractices

C++ 언어 동작 방식에 대한 의문을 테스트 코드로 검증하는 저장소입니다.

## 진행 방식

1. C++ 동작 방식에 대한 의문을 이슈/문서로 남긴다.
2. GoogleTest로 재현 가능한 테스트를 작성한다.
3. CI에서 테스트를 실행해 결과를 확인한다.
4. 테스트가 통과할 때까지 구현/검증 코드를 수정한다.
5. 통과 후 브랜치를 머지한다.

## 테스트 스타일 가이드 (BDD)

- 테스트 이름은 **한글 문장형**으로 작성합니다.
- 테스트 본문은 **주어진 상황 / 행동 / 기대 결과** 흐름으로 구성합니다.
- 의문 하나당 테스트 시나리오 하나를 기본 원칙으로 합니다.

예시:

- `값 객체를 const 참조로 전달하면 복사 생성자가 호출되지 않는다`


## CI

- `.github/workflows/cpp-tests.yml`에서 push / pull request마다 CMake 빌드와 테스트를 실행합니다.
- CI 연결 확인이 필요할 때는 의도적으로 실패하는 테스트를 임시로 추가해 파이프라인 실패를 검증할 수 있습니다.

## 로컬 실행

```bash
cmake -S . -B build
cmake --build build
ctest --test-dir build --output-on-failure
```
